# PayPal SDK — Vendor Setup

The PayPal PHP Server SDK requires a `vendor/` directory that is **generated by Composer**.
This directory is NOT included in the plugin zip — you must create it once.

## One-time setup (run on your development machine)

```bash
# From your Joomla root:
cd plugins/alfa-payments/paypal/libraries

# Install the SDK and all dependencies into vendor/
composer install --no-dev --optimize-autoloader
```

This downloads and installs:
- `paypal/paypal-server-sdk` v2.2.0 — the main SDK (675 files)
- `apimatic/core` — HTTP request pipeline, auth, retry logic
- `apimatic/core-interfaces` — interfaces for above
- `apimatic/unirest-php` — HTTP transport (cURL-based)

## After installing

Commit the `vendor/` directory to your repository:

```bash
git add vendor/
git commit -m "Add PayPal SDK vendor dependencies"
```

Then ship it inside your plugin zip. **The production Joomla server does NOT need Composer.**

## Directory layout after install

```
libraries/
├── composer.json       ← dependency definition (this is what you edit for version bumps)
├── composer.lock       ← generated by composer install (commit this too)
└── vendor/             ← generated by composer install (commit this too)
    ├── autoload.php    ← loaded by services/provider.php
    ├── composer/
    ├── apimatic/
    │   ├── core/
    │   ├── core-interfaces/
    │   └── unirest-php/
    └── paypal/
        └── paypal-server-sdk/
            └── src/
```

## Updating the SDK version

To upgrade to a newer SDK version:

```bash
cd plugins/alfa-payments/paypal/libraries
composer update paypal/paypal-server-sdk --no-dev --optimize-autoloader
git add vendor/ composer.lock
git commit -m "Upgrade PayPal SDK to x.x.x"
```

## Why not spl_autoload_register like the Klarna plugin?

The Klarna SDK had zero external dependencies — we replaced Guzzle with Joomla's
built-in `HttpFactory`. The PayPal SDK depends on `apimatic/core`, a full HTTP/auth/retry
framework that cannot be replaced with Joomla's HTTP classes. It must be shipped as-is
via Composer's vendor directory.
