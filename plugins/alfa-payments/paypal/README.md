# Alfa Commerce — PayPal Payment Plugin

Official PayPal payment gateway for [Alfa Commerce](https://github.com/Easylogic-CO-LP/Alfa-Commerce) using the [PayPal PHP Server SDK v2](https://github.com/paypal/PayPal-PHP-Server-SDK) (Orders API v2 + Payments API v2).

No JavaScript required on the storefront. Pure server-side redirect flow.

---

## Requirements

| Requirement | Version |
|---|---|
| PHP | ≥ 8.1 |
| Joomla CMS | 5.x or 6.x |
| Alfa Commerce | compatible version |
| Composer | Required once on dev machine only |
| PayPal Business Account | sandbox or live |

---

## ⚠️ Before installing — run Composer once

```bash
cd plugins/alfa-payments/paypal/libraries
composer install --no-dev --optimize-autoloader
```

Then include the `vendor/` directory in your plugin zip. See `libraries/README.md` for details.

---

## Installation

1. Run Composer in `libraries/` as above
2. Install the plugin zip via Joomla Extensions → Install
3. Enable the plugin via Joomla Extensions → Plugins → search "PayPal"
4. Configure in plugin settings: Client ID, Client Secret, Mode, Intent

---

## Configuration

| Setting | Description |
|---|---|
| Client ID | PayPal REST API Client ID (from developer.paypal.com) |
| Client Secret | PayPal REST API Client Secret |
| Mode | `sandbox` for testing, `live` for production |
| Intent | `CAPTURE` or `AUTHORIZE` (see below) |

Get your credentials at: https://developer.paypal.com/dashboard/applications

## Payment Flow

```
Customer clicks Pay with PayPal
         │
         ▼
onOrderAfterPlace()
  createOrder() via SDK
  Save PayPal order_id → PENDING
         │
         ▼
onOrderProcessView() [visit 1]
  getOrder() to retrieve approve URL
  setRedirectUrl($approveUrl)
         │
         ▼
Customer on PayPal's page
  Logs in, selects payment method, approves
         │
         ▼  PayPal appends ?token=ORDER_ID to your return_url
onOrderProcessView() [visit 2]
  captureOrder($token)   [CAPTURE intent] → COMPLETED
    OR
  authorizeOrder($token) [AUTHORIZE intent] → AUTHORIZED
         │
         ▼
onOrderCompleteView()
  Thank-you page
```

**No setState / getState needed.** Alfa Commerce carries `order_id` in the URL. PayPal appends `?token` to the same URL on redirect back. Both arrive together in visit 2.

---

## CAPTURE vs AUTHORIZE

### CAPTURE (default, recommended for digital goods)

Customer pays immediately on approval. Admin can only refund after the fact.

Admin actions available: **Refund**

### AUTHORIZE (recommended for physical goods)

Funds are reserved but not collected. Admin captures when goods are shipped.
PayPal authorizations expire after **29 days** (honor period is first 3 days).

Admin actions available: **Capture**, **Void** (before capture), **Refund** (after capture)

---

## Admin Actions

| Status | Available Actions |
|---|---|
| PENDING | View Details, View Logs |
| AUTHORIZED | View Details, View Logs, **Capture**, **Void** |
| COMPLETED | View Details, View Logs, **Refund** |
| CANCELLED | View Details, View Logs |
| REFUNDED | View Details, View Logs |

## PayPal Developer Dashboard Setup

1. Create an app at https://developer.paypal.com/dashboard/applications
2. Note Client ID and Client Secret (sandbox and live are different)
3. Add a webhook URL for live payments:
   ```
   https://yourshop.gr/index.php?option=com_alfa&task=payment.notify&plugin=paypal
   ```
   Subscribe to: `PAYMENT.CAPTURE.COMPLETED`, `PAYMENT.CAPTURE.REFUNDED`, `PAYMENT.CAPTURE.DENIED`

## Key Difference vs Klarna Plugin

| | Klarna | PayPal |
|---|---|---|
| External deps | Zero (uses Joomla HttpFactory) | Composer vendor/ required |
| Autoloading | `spl_autoload_register()` | `require_once vendor/autoload.php` |
| Amounts | Minor units (cents) as integers | Decimal strings (`"99.00"`) |
| Return token | `?sid=` (HPP session ID) | `?token=` (PayPal order ID directly) |
